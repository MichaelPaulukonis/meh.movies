## 1. Core Philosophy: "Vibe-First" Discovery

The fundamental architectural decision was to shift from **Deterministic Search** (Genre/Year) to **Semantic Discovery** (Mood/Vibe). This was achieved using an **Offline Enrichment / Online Inference** pattern.

* **Decision:** Pre-enriching the database using Claude 3.5 Haiku.
* **Trade-off:** We traded **Real-time Accuracy** for **Runtime Performance**. By "pre-baking" Gen-Z tags and vibes into SQLite, the runtime engine doesn't need to analyze plot summaries on the fly; it simply matches user moods to pre-calculated thematic markers.

## 3. Implicit Trade-offs (Hidden Decisions)

### Stochastic vs. Precise Search

* **Current:** The engine samples 20 movies and asks the LLM to pick the best match.

* **Trade-off:** We prioritize **Diversity** (users see different things every time) over **Precision** (mathematically "best" result). This prevents the "recommendation bubble" common in traditional algorithms.

### Tiered LLM Strategy

* **Background (Haiku):** Cheap/Fast for bulk enrichment of thousands of records.

* **Frontend (Sonnet):** High-intelligence for nuanced user interaction.
* **Outcome:** Optimized cost-performance ratio (approx. $0.0004 per movie enriched).

### Security vs. Friction

* **Approach:** Basic string sanitization and strict System Role guardrails.

* **Trade-off:** We kept the UX frictionless (no auth/heavy filtering) at the cost of vulnerability to sophisticated prompt injections.

## 4. Technical Debt & Evolution

### The "Tailwind Vernacular" Problem

The project relies heavily on raw utility classes. A major piece of tech debt is the lack of **Semantic Tokens**.

* *Lesson learned:* Tailwind is excellent for a POC, but for a multi-theme production app, defining a vernacular (e.g., `text-brand-primary` instead of `text-purple-500`) is mandatory to avoid maintenance friction.

### Vendor Neutrality

Post-MVP, we are migrating to **OpenRouter**.

* **Reasoning:** To prevent vendor lock-in and enable A/B testing of newer models (e.g., Llama 3 or DeepSeek) against the current Claude-based benchmark.
